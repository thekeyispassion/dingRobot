# 会议室管理 — 操作手册

## 数据来源

读写项目根目录 `data/` 中的 3 个 markdown 文件。操作方式：Read 工具读文件，Edit 工具修改文件。

**⚠️ 日志记录：** 每次写操作（添加/修改/停用/启用会议室）完成后，必须在 `data/操作日志.md` 表格末尾追加一行：
```
| 时间 | 操作类型 | 操作人 | 目标 | 详情 |
```
- 操作类型：添加会议室 / 修改会议室 / 停用会议室 / 启用会议室
- 详情：具体变更（如 "名称→新名称, 容量→50" 或 "状态→维护中，取消 2 条预约"）

| 文件 | 本模块用途 |
|------|-----------|
| `data/会议室信息.md` | 增删改查会议室 |
| `data/预约记录.md` | 停用/改设备前检查受影响用户 |
| `data/管理员.md` | 敏感操作权限校验——规则见 `manage-admin/SKILL.md` |

> 管理员账号的添加/删除由 `manage-admin` Skill 负责。

---

## 权限模型

| 级别 | 控制方式 | 适用操作 |
|------|---------|---------|
| **开放** | 任何人可操作 | 列出会议室、启用会议室、修改备注 |
| **协商确认** | 查受影响用户 → 通知 → 收集回复 → 执行 | 停用会议室、修改设备 |
| **管理员** | 双重匹配校验（见 `manage-admin/SKILL.md`） | 添加会议室、修改名称/楼栋/楼层/容量 |

---

## 触发规则

| 场景 | 关键词/意图 | 执行操作 |
|------|------------|---------|
| 查看所有会议室 | 全部会议室、列出房间、房间列表、有哪些房间 | 操作1: 列出所有会议室 |
| 添加会议室 | 添加、新增、加一间、增加 | 操作2: 添加会议室 |
| 修改会议室 | 修改、改一下、更新、调整 | 操作3: 修改会议室 |
| 停用会议室 | 停用、禁用、关闭、暂停使用、坏了、报修 | 操作4: 停用会议室 |
| 启用会议室 | 启用、恢复、重新开放 | 操作5: 启用会议室 |

---

## 可用操作

### 操作1: 列出所有会议室

**权限：开放**

Read `data/会议室信息.md` → 默认全部（含维护中），可选仅筛状态=可用 → 按楼栋/楼层/名称排序。

---

### 操作2: 添加会议室

**权限：管理员**（双重匹配校验）

1. Read `data/管理员.md` → 双重匹配校验 → 非管理员拒绝
2. Read `data/会议室信息.md` → 重名检查
3. Edit `data/会议室信息.md` 表格末尾追加：ID=最大ID+1、名称、楼栋、楼层、容量、设备、状态=可用、备注

---

### 操作3: 修改会议室

**权限：按字段分级**

| 字段 | 权限 | 控制方式 |
|------|------|---------|
| 备注 | 开放 | 直接 Edit |
| 设备 | 协商 | Read `data/预约记录.md` 查今天预约 → 通知受影响用户 → Edit |
| 名称/楼栋/楼层/容量 | 管理员 | 双重匹配校验 → 改名需查重 → Edit |

不可通过此操作修改状态字段——停用/启用走操作4/5。

---

### 操作4: 停用会议室

**权限：协商确认**（任何人可触发）

1. Read `data/预约记录.md`，筛选 会议室=目标、日期≥今天、状态=已确认
2. 0条 → Edit `data/会议室信息.md` 状态列改为「维护中」
3. 有预约 → 列出受影响的人和时间 → 触发者确认后逐个通知「取消还是保留?」
4. 取消→Edit 预约状态=已取消；保留/超时→不动
5. Edit 会议室状态=维护中，汇总结果

**核心原则：** 不替别人做决定。

---

### 操作5: 启用会议室

**权限：开放**

Read `data/会议室信息.md` 确认房间存在 → 状态≠已启用→拒绝 → Edit 状态列改为「可用」。
