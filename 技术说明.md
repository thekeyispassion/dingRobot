# OpenClaw 云服务器对接钉钉群完整部署教程

## 整体方案说明

OpenClaw 对接钉钉**必须用「企业内部机器人应用」**（普通群自定义 Webhook 机器人只能发消息、无法接收 @消息给 Agent），完整链路：

钉钉群 @机器人 → 钉钉开放平台推送消息到公网 OpenClaw 服务 → OpenClaw Agent 调用大模型 + SQLite 技能 → 结果回流钉钉群

前置条件：

1. 云服务器公网 IP / 域名，**放行端口**（默认 OpenClaw 端口 3000，安全组入站开放 3000 TCP）
2. 钉钉企业 / 组织账号（个人无企业无法创建内部机器人）
3. 服务器已装好 OpenClaw、配置好 LLM API、SQLite 技能正常运行

## 一、服务器端：安装钉钉通道插件（核心第一步）

OpenClaw 原生不带钉钉通道，需要安装官方钉钉插件

### 1. 登录云服务器 SSH，执行插件安装

bash



运行





```
# 方式1（推荐npm包）
openclaw plugins install @soimy/dingtalk

# 方式2（源码仓库安装，失败时用）
openclaw plugins install https://github.com/soimy/openclaw-channel-dingtalk.git

# 验证插件是否安装成功
openclaw plugins list
```

列表出现 `dingtalk` 即安装完成。

### 2. 进入 OpenClaw 通道配置面板

bash



运行





```
openclaw config
```

菜单选择：`Channels (Link WhatsApp/Telegram/etc and defaults)` → 选择 `DingTalk (钉钉)`

Channels配置菜单

钉钉参数录入界面

配置项先空着，**先去钉钉开放平台拿 Client ID/Client Secret**，回来再填。

## 二、钉钉开放平台：创建企业机器人应用

### 步骤 1：登录钉钉开发者后台

打开 [https://open-dev.dingtalk.com/](https://link.wtturl.cn/?target=https%3A%2F%2Fopen-dev.dingtalk.com%2F&scene=im&aid=497858&lang=zh)，用企业管理员钉钉扫码登录

1. 左侧「应用开发」→「钉钉应用」→ **立即创建**

   创建钉钉智能应用

2. 填写机器人名称（如 OpenClaw 智能助手）、图标、简介，点击创建

3. 创建成功后**立刻复制保存 Client ID、Client Secret**（只展示一次，丢失只能重置）

   获取ClientID和Secret

### 步骤 2：给应用添加「机器人」能力

1. 应用详情页 → 左侧「添加应用能力」→ 找到**机器人**，点击添加

2. 机器人配置页：

   - 消息接收模式：选择 **Stream 模式**（适配 OpenClaw 长连接）
   - 机器人名称、头像和前面保持一致，保存

   消息接收模式选Stream

### 步骤 3：开通机器人必需权限（不配置无法收发消息）

左侧「开发配置」→「权限管理」，搜索并申请开通以下权限：

1. `Chat.Message.Send` 发送群 / 单聊消息
2. `Contact.Read` 读取通讯录成员（识别 @用户）
3. `Card.Streaming.Write` 流式 AI 卡片（可选，流式回复效果更好）
4. `Card.Instance.Write` 消息卡片发送
5. `Robot.Manage` 机器人基础能力权限

申请全部权限，等待审批（企业内部应用秒过）。

权限管理页面

### 步骤 4：发布应用版本（关键！不发布群里搜不到机器人）

1. 左侧「版本管理与发布」→「创建新版本」

2. 版本描述随便填，提交发布，选择

   企业内全员可见

   创建新版本发布

## 三、回到 OpenClaw，完成钉钉通道参数填写

重新执行 `openclaw config` → Channels → DingTalk，依次填入：

1. Client ID (AppKey)：钉钉应用页面复制

2. Client Secret (AppSecret)：钉钉应用密钥

3. Corp ID：钉钉企业 ID（开放平台首页 - 企业信息查看）

4. 交互式卡片：可选 Yes（流式回复）

5. 单聊策略：`open - anyone can DM`（允许私聊机器人）

6. 群消息策略：`open`（所有内部群可 @调用）

7. 媒体白名单：填你的云服务器公网 IP

   

   全部填完保存配置，退出配置面板。

## 四、云服务器放行端口、后台常驻启动 OpenClaw

### 1. 服务器安全组放行端口

云服务器控制台→防火墙 / 安全组，**入站规则开放 3000 端口 TCP**，来源 0.0.0.0/0（钉钉平台会主动连接你的服务）

### 2. 后台常驻运行 OpenClaw（防止 SSH 断开停止）

用 pm2 托管进程：

bash



运行





```
# 安装进程托管
npm install pm2 -g

# 后台启动OpenClaw
pm2 start "openclaw start" --name openclaw-ding
# 设置开机自启
pm2 startup
pm2 save

# 查看运行日志，排查钉钉连接报错
pm2 logs openclaw-ding
```

日志出现 `DingTalk channel connected` 代表和钉钉平台长连接成功。

## 五、钉钉群添加机器人、测试对话

1. 打开你的**企业内部钉钉群**（外部群无法添加企业机器人）

2. 群右上角「...」→ 智能群助手 → 添加机器人

3. 搜索你刚才发布的机器人名称，选中添加进群

   群内添加机器人

### 测试调用流程

在群内发消息：`@OpenClaw智能助手 查询数据库里的用户数据`

数据流自动走你最开始的链路：

钉钉群消息 → OpenClaw Agent → 大模型语义解析提取关键词 → Agent 调用 SQLite 技能查询 → 组装结果返回钉钉群

## 六、常见报错排查

1. 群里搜不到机器人
   - 未发布应用新版本；群是外部群，必须企业内部群；权限未全部开通
2. @机器人无任何回复
   - 服务器 3000 端口未放行；OpenClaw 进程挂了（pm2 logs 看日志）；ClientID/Secret 填错；消息接收模式选错（必须 Stream）
3. 能收到消息，但无法操作 SQLite 数据库
   - OpenClaw 技能路径权限不足；SQLite 文件读写权限 chmod 777 *.db
4. 大模型无响应
   - 检查 OpenClaw 内 LLM API Key 配置，用 TUI 面板单独测试大模型连通性
