# 钉钉 AI 表格部署 — 操作手册

## 概述

本 Skill 内置了项目所需的 AI 表格结构定义（3 张表）。部署文档里只需写一句「执行 deploy-ai-table skill」，Agent 按手册中定义的表结构，通过 MCP 工具在钉钉中创建对应 Base、数据表和字段。

## 核心约束

1. **以下表定义为唯一数据源** — 不允许凭空编造字段
2. **必须先搜索再创建** — 同名 Base 可能已存在
3. **关联字段用 link 类型** — 引用另一张表的字段，确保目标表先创建
4. **输出完整结果** — 返回 Base ID、表 ID、字段列表

**创建顺序（关联依赖）：**
1. 先创建 `会议室信息`（无外部关联，被预约记录引用）

3. 最后创建 `预约记录`（含 link→会议室信息）

**主键说明：** 钉钉 AI 表格的每张 Sheet 自动生成记录 ID，无需手动创建。

## 工作流

### Step 1: 搜索或创建 Base

1. 用 MCP 工具搜索名为 **「AI会议室预约助手」** 的 Base
2. 判断结果：

| 搜索结果 | 处理 |
|---------|------|
| 已存在 | 询问用户：清空重建 / 复用已有 / 新建（名称加时间戳后缀） |
| 不存在 | 创建 Base：名称=AI会议室预约助手，描述=学院会议室预约系统数据 |

3. 记录 Base ID

### Step 2: 创建数据表

按上述创建顺序，对每张表执行：

1. **创建 Sheet** — 在目标 Base 中创建数据表，传入 base_id 和表名
2. **添加字段** — 按表中字段顺序逐个添加：
   - `text` / `number` / `date` / `time`：直接创建
   - `single_select`：先创建字段，再添加选项列表
   - `link`：指定关联的目标表（确保目标表已先创建）
3. **标记必填** — 属性为「必填」的字段设为必填
4. 记录 Sheet ID 和字段 ID

### Step 3: 输出结果

完成后输出结构化结果和人类可读摘要：

```
✅ 钉钉 AI 表格部署完成！

Base: AI会议室预约助手
🔗 https://table.dingtalk.com/base/<base_id>

数据表（3 个）:
  • 会议室信息  — 7 字段（含状态单选: 可用/维护中）
  • 管理员      — 4 字段（角色默认 admin）
  • 预约记录    — 8 字段（含会议室关联 + 状态单选: 已确认/已取消）

请在钉钉中打开 Base URL 确认表结构。
```

```json
{
  "success": true,
  "base": {"name": "AI会议室预约助手", "id": "<base_id>"},
  "tables": [
    {"name": "会议室信息", "fields": 7},
    {"name": "管理员",     "fields": 4},
    {"name": "预约记录",   "fields": 8}
  ]
}
```

## 常见问题

| 问题 | 处理 |
|------|------|
| Base「AI会议室预约助手」已存在 | 询问：清空重建 / 复用 / 新建（加时间戳后缀） |
| 部分字段创建失败 | 已完成不回滚，结果中区分 created/failed，给出重试建议 |
| 创建中途中断 | 下次执行时先搜索已有 Base，增量补齐缺失的表和字段 |
| MCP 工具未连接 | 提示检查钉钉 AI Table MCP Server 配置 |
| 权限不足 | 提示在钉钉中申请 Base 编辑权限 |

## MCP 工具发现

执行前用 ToolSearch 搜索钉钉 AI 表格工具：

- 关键词：`dingtalk`、`ai_table`、`base`、`sheet`、`field`
- 常见工具名：`create_base`、`create_sheet`、`add_field`、`search_base`

未找到时提示：

> 未发现钉钉 AI 表格 MCP 工具，请确认：
> 1. 钉钉 AI 表格 MCP Server 是否已安装并启动
> 2. settings.json / .mcp.json 中是否配置了对应 MCP
