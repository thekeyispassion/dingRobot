# 钉钉 AI 表格 — 建表配置

## 连接方式

**MCP Server：** 钉钉 AI 表格（streamable-http）  
**目标类型：** Base  
**Base 名称：** AI会议室预约助手  
**Base 描述：** 学院会议室预约系统数据  
**关键工具：** search_bases, create_base, create_table, create_fields, update_view

---

## 数据表定义

### 表 1: 会议室信息

| 字段名 | 钉钉类型 | 属性 |
|--------|---------|------|
| 会议室名称 | text | 必填 |
| 状态 | single_select | 必填，选项: 可用, 维护中 |
| 楼栋 | text | 必填 |
| 楼层 | number | 必填 |
| 容量 | number | 必填 |
| 设备 | text | 可选 |
| 备注 | text | 可选 |

主键：钉钉自动生成记录 ID，无需手动创建。

---

### 表 2: 管理员

| 字段名 | 钉钉类型 | 属性 |
|--------|---------|------|
| 用户ID | text | 必填（降级场景可为空字符串） |
| 用户姓名 | text | 必填 |
| 角色 | text | 默认 admin |
| 创建时间 | text     | 必填 |

---

### 表 3: 预约记录

| 字段名 | 钉钉类型 | 属性 |
|--------|---------|------|
| 会议室 | link | 关联→会议室信息, 必填 |
| 用户ID | text | 必填 |
| 用户姓名 | text | 必填 |
| 日期 | date | 必填 |
| 开始时间 | text | 必填 |
| 结束时间 | text | 必填 |
| 状态 | single_select | 必填，选项: 已确认, 已取消 |
| 创建时间 | text | 必填 |

---

## 创建顺序

按关联依赖排序——被引用的表先创建：

1. **先** `会议室信息`（无外部关联，被预约记录 link 引用）
2. **再** `管理员`（无外部关联）
3. **最后** `预约记录`（含 link→会议室信息）

---

## 建表后操作

### 调整预约记录视图列顺序

钉钉 AI 表格创建 link 字段时自动排到最后一列。通过 `update_view` 把「会议室」link 字段从末尾移到首列之后：

```
visibleFieldIds 顺序:
  1. [首列索引]
  2. 用户姓名
  3. 会议室（link）
  4. 用户ID
  5. 日期
  6. 开始时间
  7. 结束时间
  8. 状态
  9. 创建时间
```

---

## 注意事项

- **传输协议：** 钉钉 MCP 使用 `streamable-http`，不是默认 SSE。不加 `transport` 字段会报 405
- **link 字段顺序：** 创建时不传 link 字段位置，建完后再用 `update_view` 调整
- **single_select 选项：** 创建字段时附带 `options` 参数，不单独追加
