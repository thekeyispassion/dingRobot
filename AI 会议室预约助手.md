#  基于OpenClaw的AI会议室预约系统(实训课题挑战版)

## 计算机学院 AI 会议室预约助手实训课题（基于 OpenClaw）

> 一句话，约一间房。 不用找管理员，不用填表单，在钉钉群里 @机器人 说句话就行。

### 一、 课题背景

传统的会议室预约流程存在明显痛点：流程繁琐（登录系统、选楼栋、填信息，10+ 次点击）、信息分散、时间冲突频发、审批等待周期长。

本课题要求基于 OpenClaw 框架，开发一个 AI 会议室预约助手。该助手以钉钉群机器人（虚拟用户）的形态，直接加入学院的钉钉群。师生有任何会议室预约需求时，只需在群里 @机器人 并用自然语言描述需求，机器人即可自动完成查询、预约、取消等操作。

### 二、 系统形态

本系统没有独立的 Web 界面。它的唯一交互入口就是钉钉群：

机器人作为一个虚拟用户加入学院钉钉群。

用户通过 @机器人 + 自然语言发起请求。

机器人理解意图后，调用后端 Skill 完成操作，并在群内回复结果。

### 三、 功能需求

#### 3.1 基础预约

用户 @机器人 说出时间、房间等需求，机器人自动完成预约并返回确认信息（房间、时段、预约 ID）。

> 示例：
>
> @预约助手 帮我约明天下午 330
>
>  → 
>
> 预约成功！信电楼330 | 5月3日 13:30-16:15 | ID: 8852

#### 3.2 空闲查询与预约总览

用户可以查询当前有哪些房间还空着，也可以查看学院会议室资源在某段时间内的整体预约情况。

> 示例1：
>
> @预约助手 现在有哪些空房间？
>
>  → 返回当前时段所有空闲会议室列表
>
> 示例2：
>
> @预约助手 明天下午各会议室的预约情况
>
>  → 返回明天下午所有房间的占用/空闲状态一览

#### 3.3 模糊时间理解

支持”下午”“傍晚”“明天的明天”等模糊或口语化的时间表达，系统能正确解析为标准时段。

#### 3.4 智能冲突处理与推荐

当目标房间已被占用时，机器人不能只报错，必须主动推荐容量相近的替代房间。

> 示例：
>
> @预约助手 明天下午 330
>
>  → 
>
> 330已满，推荐317（容量相近），是否预约？

#### 3.5 个人预约查询与取消

用户可以随时查询自己的预约记录，也可以取消自己的预约。

#### 3.6 权限隔离

机器人通过钉钉的用户身份自动识别”谁在说话”，每个人只能查看和取消自己的预约。

#### 3.7 边界友好提示

对不存在的房间号、非开放时段、格式无法理解的请求，给出友好的自然语言提示而非报错。

### 四、 技术思路

| 层次   | 职责                         | 大致方案                             |
| ------ | ---------------------------- | ------------------------------------ |
| 交互层 | 接收钉钉群消息，回复结果     | OpenClaw Gateway + 钉钉机器人接入    |
| 理解层 | 解析自然语言，提取意图与参数 | 国产大模型 API（如 Qwen / DeepSeek） |
| 业务层 | 查询、预约、取消、推荐等逻辑 | OpenClaw Skill（Python 脚本）        |
| 数据层 | 存储会议室与预约记录         | SQLite                               |

### 五、 部署方案建议（经济实惠）

OpenClaw 本身免费开源，主要成本来自云服务器和大模型 API 调用。国内主流云厂商均提供了预装 OpenClaw 的一体机镜像，学生无需自己折腾环境，一键即可部署。以下是推荐的低成本方案：

| 方案                 | 配置                 | 价格                    | 说明                                 |
| -------------------- | -------------------- | ----------------------- | ------------------------------------ |
| 阿里云轻量应用服务器 | 2核2G + OpenClaw镜像 | 约 38 元/年（9.9元/月） | 一键部署，选择 OpenClaw 应用镜像即可 |
| 腾讯云 Lighthouse    | 2核4G + OpenClaw镜像 | 188 元/年（送 3 个月）  | 同样支持 OpenClaw 应用镜像一键部署   |

### 六、 验收标准

验收时不需要在学院正式钉钉群中进行。学生自建一个小型团队钉钉群，将预约机器人引入该群即可演示和测试。

1.在自建钉钉群中成功接入机器人，@机器人 后能正常响应。

2.能成功演示：基础预约、空闲查询与预约总览、模糊时间解析、冲突推荐、个人预约查询/取消 等核心场景。

3.提交一份包含至少 15 个测试用例的测试报告，覆盖正常流程与边界情况，通过率 100%。
