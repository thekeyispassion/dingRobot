# 钉钉 AI 表格 — 镜像同步配置

## 连接方式

**MCP Server：** 钉钉 AI 表格（streamable-http）

**工具：** search_records, create_record, update_record, delete_record, get_base, get_tables

## 日志 → AI 表格操作翻译

### 预约

```
1. Read AI 表格「会议室信息」全部记录 → 建立映射表 {名称 → recordId}
2. 从日志「详情」中提取：日期=、开始=、结束=
3. 从日志「目标」中提取房间名
4. 从日志「操作人」中提取用户姓名、userId
5. 查 AI 表格「预约记录」是否已存在匹配记录：
   筛选 用户姓名 + 日期 + 开始时间 + 结束时间 + 会议室 recordId 均匹配
   → 已存在：跳过（幂等，防止重复同步）
   → 不存在：create_record
6. create_record「预约记录」:
   会议室 = 映射表[房间名]（link字段传 AI 记录 ID）
   用户ID = 提取的 userId
   用户姓名 = 提取的姓名
   日期 = 提取的日期
   开始时间 = 提取的开始时间
   结束时间 = 提取的结束时间
   状态 = 已确认
   创建时间 = 日志「时间」
```

### 取消预约

```
1. 从日志「详情」中提取：日期=、开始=、结束=
2. 从日志「目标」获取房间名，「操作人」获取用户姓名
3. 查 AI 表格「预约记录」，筛选 用户姓名 + 日期 + 开始时间 + 结束时间 匹配
   → 已取消：跳过（幂等）
   → 已确认：update_record 状态 = 已取消
```

> markdown 的预约 ID 和 AI 表格的记录 ID 是两个体系。用姓名+日期+开始时间+结束时间组合匹配。日志中的 预约ID 仅作参考。

### 添加会议室

```
1. create_record「会议室信息」: 从日志「详情」提取各字段
2. 记录新 AI 表格记录 ID 到映射表
```

### 修改会议室

```
1. 从日志「目标」获取房间名
2. 查 AI 表格「会议室信息」，筛选 会议室名称 = 目标
3. update_record: 更新日志「详情」中提到的字段
```

### 停用/启用会议室

```
1. 从日志「目标」获取房间名
2. 查 AI 表格「会议室信息」，筛选 会议室名称 = 目标
3. update_record: 状态 = 维护中 / 可用
```

### 添加管理员

```
1. create_record「管理员」: 从日志「详情」提取 用户ID、用户姓名、角色
```

### 删除管理员

```
1. 从日志「详情」提取 用户姓名
2. 查 AI 表格「管理员」，筛选 用户姓名 = 目标
3. 如有多条，再比对 userId
4. delete_record
```

### 补全 userId

```
1. 从日志「详情」提取 用户姓名 + 新 userId
2. 查 AI 表格「管理员」，筛选 用户姓名 = 目标
3. update_record: 用户ID = 新 userId
```

## 首次同步（全量）

如果 `.sync-marker` 不存在、最后同步时间为 epoch（`1970-01-01T00:00:00`）、或 AI 表格中 3 张 Sheet 均为空，走全量流程：

1. 清空 AI 表格中 3 张 Sheet 的全部记录
2. Read 3 个 markdown 文件
3. 先导入会议室信息 → 建立映射表
4. 再导入管理员
5. 最后导入预约记录（用映射表翻译房间名→link）
6. 写入 `.sync-marker`，时间戳 = 最后一条日志的时间

---

## 注意事项

- **同步顺序固定：** 会议室信息 → 管理员 → 预约记录。预约记录依赖会议室信息的 link 字段
- **ID 不通用：** markdown 的 ID 和 AI 表格的记录 ID 是两个体系。匹配靠「名称+时间」等业务字段
- **同名房间不会变：** 房间名称是业务唯一标识。即使修改了房间名，日志里会有"修改会议室"记录，用旧名称匹配后再改
- **同步幂等：** 同一条日志重复执行不会造成重复数据——用业务字段匹配后 update 而非盲目 create
