# 镜像同步 — 操作手册

## 概述

本地 `data/` markdown 文件是唯一数据源。本 Skill 负责将变更增量同步到外部镜像。

**核心机制：日志驱动增量同步。** 读 `data/操作日志.md`，找到上次同步之后的新日志条目，逐条翻译为镜像操作。

**同步标记：** `data/.sync-marker` 文件中记录每个镜像最后同步到的日志时间戳。每次同步完成后更新。

## 支持的镜像

| 镜像 | 配置文件 | 用途 |
|------|---------|------|
| 钉钉 AI 表格 | `mirror-sync/钉钉AI表格-配置.md` | 在线查看 + 链接分享 |

> 新增通道时，在 `mirror-sync/` 下新增配置文件即可。

---

## 工作流

### Step 1: 读取同步标记

Read `data/.sync-marker`，获取每个镜像的最后同步时间：

```
钉钉AI表格: 2026-07-18T15:30:00
```

如文件不存在 → 全部日志尚未同步 → 从第 1 条开始。

### Step 2: 判断是否需要全量同步

以下任一条件满足时，**跳过增量，直接走首次全量同步**（见下方「首次同步」章节）：

1. `.sync-marker` 不存在
2. 该通道的最后同步时间为 epoch（`1970-01-01T00:00:00`）
3. 该通道对应的 AI 表格/镜像中**所有表均为空**（无任何记录）

> 条件 3 覆盖的场景：deploy-ai-table 已建表但从未同步数据，marker 已存在但 data/ 有种子数据。

### Step 3: 读取新日志

Read `data/操作日志.md`，筛选时间 > 上次同步时间的所有行。

0 条新增 → "所有数据已同步，无需操作。" → 结束。

### Step 4: 逐条翻译并执行

对每条新日志，根据 `操作类型` 查对应镜像的配置文件，翻译为具体操作。

翻译规则由各镜像配置文件定义。通用映射：

| 日志操作类型 | 影响表 | 镜像动作 |
|------------|--------|---------|
| 预约 | 预约记录 | 新增行 |
| 取消预约 | 预约记录 | 更新状态=已取消 |
| 添加会议室 | 会议室信息 | 新增行 |
| 修改会议室 | 会议室信息 | 更新指定列 |
| 停用会议室 | 会议室信息 | 更新状态=维护中 |
| 启用会议室 | 会议室信息 | 更新状态=可用 |
| 添加管理员 | 管理员 | 新增行 |
| 删除管理员 | 管理员 | 删除行 |
| 补全userId | 管理员 | 更新用户ID列 |

**日志字段解析：** 日志的 `详情` 列使用 `key=value, key=value, ...` 格式。操作人和目标列也可提取结构化信息：

| 日志列 | 格式 | 示例 |
|--------|------|------|
| 操作人 | `姓名 (userId)` | `张三 (user001)` |
| 目标 | 纯文本 | `信电楼330` |
| 详情 | `key1=val1, key2=val2, ...` | `日期=2026-07-20, 开始=14:00, 结束=16:00, 预约ID=5` |

各操作类型的详情 key 定义见对应核心 Skill 的「日志记录」节。通道配置文件中的翻译规则引用这些 key 来提取数据。

### Step 5: 更新同步标记

Edit `data/.sync-marker`，将该镜像的时间戳更新为最后一条已同步日志的时间。

### Step 6: 输出结果

```
✅ 钉钉 AI 表格已同步
   新增: 预约 2 条, 会议室 1 间
   更新: 取消 1 条, 停用 1 间
   详细: https://table.dingtalk.com/base/<id>
```

## 触发规则

| 场景 | 关键词 | 行为 |
|------|-------|------|
| 手动同步 | "同步到 AI 表格"、"更新镜像"、"同步数据" | 执行增量同步 |
| 写操作后自动 | 每次写操作完成 | 同上                                              |
| 查看同步状态 | "同步状态"、"镜像状态" | Read `data/.sync-marker` → 输出各镜像最后同步时间 |

---

## .sync-marker 格式

```
钉钉AI表格: 2026-07-18T15:30:00
其他通道: 2026-07-18T14:00:00
```

每行一个镜像，格式：`镜像名: ISO时间戳`。

---

## 新增通道

在 `mirror-sync/` 下新增配置文件，包含：
1. 连接方式
2. 日志操作类型 → 镜像操作的翻译规则
3. link 字段映射（如何处理跨表关联）
4. 同步标记名称

参考 `mirror-sync/钉钉AI表格-配置.md`。
