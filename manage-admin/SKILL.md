# 管理员账号管理 — 操作手册

## 概述

管理 `data/管理员.md`。核心难点：用户说"加张三"时只有姓名没有 userId——通过钉钉通讯录 MCP 自动解析。

**⚠️ 日志记录：** 每次写操作（添加/删除管理员、补全 userId）完成后，必须在 `data/操作日志.md` 表格末尾追加一行：
```
| 时间 | 操作类型 | 操作人 | 目标 | 详情 |
```
- 时间：当前时间 ISO 格式
- 操作人：`姓名 (userId)`，如 `张三 (user001)`
- 目标：被操作的管理员姓名
- 详情：**严格按以下格式，mirror-sync 依赖此格式解析**

| 操作类型 | 详情格式 | 示例 |
|---------|---------|------|
| 添加管理员 | `userId=xxx, 姓名=xxx, 角色=admin, 来源=通讯录` 或 `来源=降级匹配` | `userId=033364..., 姓名=张三, 角色=admin, 来源=通讯录` |
| 删除管理员 | `姓名=xxx, userId=xxx` | `姓名=张三, userId=033364...` |
| 补全userId | `姓名=xxx, userId=xxx` | `姓名=张三, userId=033364...`

## 可用操作

### 操作1: 添加管理员

**权限：管理员**（双重匹配校验）

1. Read `data/管理员.md` → 双重匹配校验 → 非管理员拒绝
2. 钉钉通讯录 MCP：`search_contact_by_key_word(目标姓名)`
   - 1人 → userId；多人 → 列出部门让选；0人 → 降级（提示风险确认后 userId=""）
3. Read `data/管理员.md` 去重（userId 匹配或 userId=""+姓名匹配）
4. Edit `data/管理员.md` 表格末尾追加：ID=最大ID+1、用户ID、用户姓名、角色=admin、创建时间=当前

---

### 操作2: 删除管理员

**权限：管理员**

1. 双重匹配校验 → Read `data/管理员.md` 找目标 → 确认 → Edit 删除行
2. 不能删最后一个管理员

---

### 操作3: 查看管理员列表

**权限：开放**

Read `data/管理员.md` → 用户ID为空的行标注 ⚠️ 降级匹配。

---

### 操作4: 补全 userId

**权限：管理员**（全自动批量）

1. Read `data/管理员.md`，找所有用户ID=""的行 → 0条则结束
2. 逐条调通讯录 MCP → 唯一匹配直接 Edit userId → 多名/搜不到跳过
3. 包括操作者自己的空 userId 一并补全
4. 汇总输出：✅ 补全 N 人 / ⚠️ 跳过 M 人

🚨 **绝对禁止用操作者自己的 userId 填入。**

---

## 权限校验规则（所有 Skill 共用）

```
Read data/管理员.md:

  第一优先：用户ID = 当前userId → ✅ 管理员

  第二优先：用户ID = "" AND 用户姓名 = 当前userName → ⚠️ 管理员（降级）
           → 命中后自动尝试通讯录 MCP 补全 userId

  都不匹配 → ❌ 非管理员
```
