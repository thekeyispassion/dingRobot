# 会议室预约助手 — 操作手册

## 数据来源

读写项目根目录 `data/` 中的 3 个 markdown 文件：

| 文件 | 用途 |
|------|------|
| `data/会议室信息.md` | 会议室基本信息（markdown 表格） |
| `data/预约记录.md` | 预约流水（markdown 表格） |
| `data/管理员.md` | 管理员白名单（markdown 表格） |

**操作方式：** Agent 用 Read 工具读文件，Edit 工具修改文件。

**⚠️ 日志记录：** 每次写操作（预约、取消）完成后，必须在 `data/操作日志.md` 表格末尾追加一行：

```
| 时间 | 操作类型 | 操作人 | 目标 | 详情 |
```
- 时间：当前时间 ISO 格式
- 操作类型：预约 / 取消预约
- 操作人：`姓名 (userId)`，如 `张三 (user001)`
- 目标：房间名
- 详情：**严格按以下格式，mirror-sync 依赖此格式解析**

| 操作类型 | 详情格式 | 示例 |
|---------|---------|------|
| 预约 | `日期=YYYY-MM-DD, 开始=HH:mm, 结束=HH:mm, 预约ID=N` | `日期=2026-07-20, 开始=14:00, 结束=16:00, 预约ID=5` |
| 取消预约 | `日期=YYYY-MM-DD, 开始=HH:mm, 结束=HH:mm, 预约ID=N` | `日期=2026-07-20, 开始=14:00, 结束=16:00, 预约ID=5` |

**外部镜像：** 本 Skill 只操作本地 data/ 文件。如需分享完整数据，附带在线表格链接即可。外部镜像的同步由 `mirror-sync` Skill 负责。

## 触发规则

| 场景 | 关键词/意图 | 执行操作 |
|------|------------|---------|
| 预约会议室 | 约、定、订、预约、帮我订、book | 解析时间+房间 → 操作4: 预约会议室 |
| 查询空闲 | 空房间、空闲、有哪些、空着 | 解析时间 → 操作1: 查询空闲房间 |
| 今日状态 | 现在谁在、当前状态、在用 | 操作2: 今日实时状态 |
| 预约日程 | 预约情况、谁约了、日程、一览 | 解析日期 → 操作3: 某天预约日程 |
| 我的预约 | 我的预约、我约了、我订了 | 操作5: 查询我的预约 |
| 取消预约 | 取消、退订、不要了 | 有记录ID直接取消，无ID先查再确认 |
| 询问房间 | 有哪些房间、会议室列表 | Read `data/会议室信息.md` |

## 可用操作

### 操作1: 查询空闲房间

Read `data/会议室信息.md`（筛选状态=可用）+ Read `data/预约记录.md`（筛选日期=目标、状态=已确认、时段重叠）→ 排除冲突房间 → 按楼栋/楼层排序返回。

时段重叠条件：开始时间 < 目标结束时间 AND 结束时间 > 目标开始时间。

---

### 操作2: 今日实时状态

获取当前日期+时刻 → Read `data/会议室信息.md`（状态=可用）+ Read `data/预约记录.md`（日期=今天、状态=已确认）→ 遍历房间判断：start≤now<end → 使用中；else → 空闲+upcoming。

---

### 操作3: 某天预约日程

Read `data/会议室信息.md`（状态=可用）+ Read `data/预约记录.md`（日期=目标、状态=已确认）→ 按房间分组，booking_count=0 → 全天可约。

---

### 操作4: 预约会议室

1. 参数校验：开始≥结束→拒绝；日期<今天→拒绝
2. Read `data/会议室信息.md`，模糊匹配房间名
3. Read `data/预约记录.md`，冲突检测（同房间+同日期+时段重叠）
4. 有冲突 → 查空闲 + 同楼栋优先+容量相近 Top3 推荐
5. 无冲突 → Edit `data/预约记录.md` 表格末尾追加一行：ID=最大ID+1、会议室、用户ID、用户姓名、日期、时间、状态=已确认、创建时间=当前

**参数：** USER_ID、USER_NAME、房间名（模糊）、日期、开始时间、结束时间

---

### 操作5: 查询我的预约

Read `data/预约记录.md` → 筛选 用户ID=目标、状态=已确认 → 按日期+时间排序。

---

### 操作6: 取消预约

Read `data/预约记录.md` 按 ID 找行 → 状态=已取消→拒绝；用户ID≠当前→权限不足 → Edit 该行，状态列改为「已取消」。

> 用户不提供 ID 时，先执行操作5 列出让用户选。

---

### 操作7: 时间解析（辅助）

纯计算，映射表：

| 中文 | 时段 |
|------|------|
| 上午 | 08:00-12:00 |
| 中午 | 12:00-14:00 |
| 下午 | 14:00-18:00 |
| 傍晚 | 18:00-21:00 |
| 晚上 | 19:00-22:00 |

日期偏移：今天=0, 明天=+1, 后天=+2

---

## 冲突检测规则

```
重叠条件: start_a < end_b AND end_a > start_b
```

不含端点——16:00 结束和 16:00 开始不冲突。
